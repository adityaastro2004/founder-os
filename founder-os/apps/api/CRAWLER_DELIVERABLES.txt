================================================================================
ARCHITECTURE & DESIGN PATTERNS
================================================================================

1. CrawlEngine (Low-Level)
   - Simple, stateless crawler
   - No external API dependencies
   - Rate limiting via Semaphore
   - robots.txt checking
   - HTML/RSS parsing with stdlib
   
2. ResearchEngine (High-Level)
   - Orchestrates full research workflow
   - Ties crawling to founder context
   - Query generation from profile
   - Relevance scoring and filtering
   - Finding storage and categorization
   
3. Factory Pattern
   - get_crawler_engine() singleton
   - get_research_engine() singleton
   - Lazy initialization
   
4. Dataclass Models
   - Type-safe data structures
   - Clear contracts between components
   - Easy serialization for API responses

5. Async/Await
   - All I/O is async (HTTP, database, file)
   - Concurrent research cycles possible
   - Non-blocking rate limiting
   - Proper connection pooling

6. Integration Points
   - memory_pages table (findings storage)
   - planner_users table (context reading)
   - MemoryManager (embedding + storage)
   - FastAPI router (API endpoints)
   - Clerk auth (JWT validation)
