# Retro — task 016: schema baseline migration (2026-07-11 → 2026-07-14)

## What happened

Production onboarding returned "API error 500" on 2026-07-11. Two stacked causes:

1. **Frontend**: the Vercel deployment was prebuilt on the founder's laptop with
   `NEXT_PUBLIC_API_URL` unset → the `/api/*` rewrite proxied to
   `http://localhost:8000` inside Vercel. Fixed same day (rebuild with the prod
   URL); Vercel is now GitHub-connected (rootDirectory corrected to
   `founder-os/apps/web` for the double-nested repo) so dashboard env vars govern
   future builds.
2. **Backend**: once traffic reached the API, the query failed on
   `founder_profiles.primary_goal_description does not exist`. Schema truth was
   fragmented across four sources (schema.sql, raw migrations/002–005 that no
   pipeline applied, two alembic revisions, and ORM-only columns). Prod DB was
   rebuilt by hand same day; task 016 then made the fix structural.

## What shipped (task 016, ADR-011)

Re-rooted frozen alembic baseline (`0000_baseline`): `alembic upgrade head` alone
bootstraps a complete, ORM-consistent DB from any of three starting states (empty /
legacy schema.sql-seeded / already at head). Raw migrations retired; schema.sql
demoted to a banner-marked secondary artifact; new `migrations` pytest tier in CI
asserts name-level ORM parity at head — the incident class now fails CI instead of
failing production. Full pipeline ran: architect (ADR-011) → executor → QA (6/6
criteria PASS, independently re-proven) → reviewer (approve-with-nits; nits 1–2
applied, 3 noted, 4 deferred).

## What slowed development (§9 loop)

- **Fragmented schema truth** was invisible until production broke — no CI step
  compared ORM metadata to any real database. Now structural (parity test at head).
- **Deploy topology knowledge lived nowhere**: prebuilt-vs-dashboard env semantics,
  the two `.vercel` link dirs, the double-nested rootDirectory trap, SSH actually
  being open. Now captured in deploy.yml comments, ADR-011 context, and this report.
- **`.env` drift broke local tooling**: a stray `GROQ_API=` key crashed
  pydantic-settings (`extra_forbidden`), so alembic/uvicorn wouldn't even start
  locally during an incident. Consider a `.env` lint (compare keys against
  `config.py`) as a future guard.
